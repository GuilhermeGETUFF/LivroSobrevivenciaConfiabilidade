15.4 UMA COVARIÁVEL

Para lidarmos bem com a saída dos modelos de regressão paramétricos e com a interpretação dos coeficientes, esta seção é dedicada ao estudo de dados com somente uma covariável.

15.4.1 UMA COVARIÁVEL QUANTITATIVA

Vamos ilustrar esse cenário com a base de dados de leucemia aguda. Para isso, vamos carregar a base de dados:

Código 
temp<-c(65,156,100,134,16,108,121,4,39,143,56,26,22,1,1,5,65)
 cens<-rep(1,17)
 log10_wbc<-c(3.36,2.88,3.63,3.41,3.78,4.02,4.00,4.23,3.73,3.85,3.97,
         4.51,4.54,5.00,5.00,4.72,5.00)
dados1<-data.frame(temp,cens,log10_wbc)

Em muitas aplicações, algumas covariáveis apresentam valores muito elevados numericamente, como renda, população, carga viral, PIB ou número de acessos. Nesses casos, é comum utilizar a transformação logarítmica da variável devido às seguintes vantagens:

  • Redução da magnitude numérica dos valores, tornando a escala mais controlada;
  • Melhora da estabilidade computacional dos algoritmos de estimação;
  • Redução de problemas de convergência em métodos iterativos;
  • Melhor condicionamento das matrizes envolvidas na estimação;
  • Diminuição da influência de valores extremamente altos;
  • Possibilidade de interpretar os efeitos em termos percentuais ou multiplicativos;
  • Redução de assimetria em variáveis fortemente assimétricas.

Assim, a transformação logarítmica frequentemente produz modelos mais estáveis, interpretáveis e adequados aos dados observados.

Para esta base de dados, consideraremos a seguinte estrutura regressora: \[\beta_0+\beta_1x_i,\] em que \(x_i\) é o valor de log10_wbc para o \(i\) paciente, com \(i=1,...,17\).

15.4.1.1 AJUSTE DE MODELOS PARAMÉTRICOS

Para ajustar cada um dos 5 modelos de regressão paramétricos vistos na disciplina, considere os seguintes comandos:

Código 
ajuste_exp<-flexsurvreg(data=dados1,                                                    Surv(temp,cens)~log10_wbc,
                        dist="exponential")
ajuste_wei<-flexsurvreg(data=dados1,                                                    Surv(temp,cens)~log10_wbc,
                        dist="weibull")
ajuste_lognorm<-flexsurvreg(data=dados1,                                                    Surv(temp,cens)~log10_wbc,
                        dist="lognormal")
ajuste_gama<-flexsurvreg(data=dados1,                                                    Surv(temp,cens)~log10_wbc,
                        dist="gamma")
ajuste_gengama<-flexsurvreg(data=dados1,                                                    Surv(temp,cens)~log10_wbc,
                        dist="gengamma")

MODELO EXPONENCIAL

Para o modelo exponencial, tem-se a seguinte saída:

Código 
tidy(ajuste_exp)
# A tibble: 2 × 5
  term      estimate std.error statistic  p.value
  <chr>        <dbl>     <dbl>     <dbl>    <dbl>
1 rate      0.000208  0.000356     NA    NA      
2 log10_wbc 1.11      0.414         2.68  0.00731

De acordo com o modelo ajustado, temos que:

  • rate = \(exp\{\hat{\beta_0}\}=0,0002\rightarrow \hat{\beta_0} = ln(0,0002) = -8,48\)
  • \(\hat{\beta_1}=1,11\)

Portanto, cada tempo de sobrevivência \(T_i\) possui a seguinte distribuição:

\[T_i\sim Exp(\hat{\alpha_i}=e^{-8,48 + 1,11x_i})\]

Com essa distribuição estimada, podemos obter todas as quantidades de interesse em sobrevivência (funções básicas e medidas-resumo)

Nesse modelo:

  • A estimativa de \(\hat{\beta_0}\) é negativa
  • A estimativa de \(\hat{\beta_1}\) é positiva

Podemos interpretar o coeficiente \(\hat{\beta_1}\) da seguinte forma:

Código 
RTM = c(
  Est = exp(-ajuste_exp$coefficients[2]),
  LI = exp(-confint(ajuste_exp))[4],
  LS = exp(-confint(ajuste_exp))[2]
)
RTM
Est.log10_wbc            LI            LS 
    0.3297904     0.1466227     0.7417795

Assim, o aumento de uma unidade no log10_wbcreduz o tempo mediano de sobrevivência em (1-0,32) = 68%

TESTE DA RAZÃO DE VEROSSIMILHANÇA

Para investigar a adequação do modelo mais complexo (gama generalizado) comparado aos demais a partir do teste TRV, considere os seguintes comandos:

Código 
TRV_exp = 2*(ajuste_gengama$loglik-ajuste_exp$loglik)
P_valor_exp = pchisq(TRV_exp,2,lower.tail=F)
TRV_wei = 2*(ajuste_gengama$loglik-ajuste_wei$loglik)
P_valor_wei = pchisq(TRV_wei,1,lower.tail=F)
TRV_lognorm = 2*(ajuste_gengama$loglik-ajuste_lognorm$loglik)
P_valor_lognorm = pchisq(TRV_lognorm,1,lower.tail=F)
TRV_gama = 2*(ajuste_gengama$loglik-ajuste_gama$loglik)
P_valor_gama = pchisq(TRV_gama,1,lower.tail=F)
Comparacao<-c("Exponencial x GG", "Weibull x GG",
           "LogNormal x GG",
           "Gama x GG")
P_valor = c(P_valor_exp,P_valor_wei,
            P_valor_lognorm,P_valor_gama)
TRV<-data.frame(Comparacao,P_valor)
TRV
        Comparacao     P_valor

2     Weibull x GG 0.009962565
3   LogNormal x GG 0.011305442

15.4.2 UMA VARIÁVEL QUALITATIVA

Para uma única variável qualitativa, vamos considerar a base de dados de malária com a variável explicativa grupos. Para isso, vamos carregar a base de dados:

Código 
temp<-c(7,8,8,8,8,12,12,17,18,22,30,30,30,30,30,30,8,8,9,10,10,14,
          15,15,18,19,21,22,22,23,25,8,8,8,8,8,8,9,10,10,10,11,17,19)
cens<-c(rep(1,10), rep(0,6),rep(1,15),rep(1,13))
grupos<-c(rep("Grupo 1",16), rep("Grupo 2",15), rep("Grupo 3",13))
dados2<-data.frame(temp,cens,grupos)

Para esta base de dados, o Grupo 1 será a referência e consideraremos a seguinte estrutura regressora: \[\beta_0+\beta_1x_{i1}+\beta_2x_{i2},\] em que:

  • \(x_i1\) é uma indicadora do rato estar no Grupo 2
  • \(x_i2\) é uma indicadora do rato estar no Grupo 3

Como a variável é qualitativa, é bem interessante começar a análise fazendo um teste de comparação de curvas de sobrevivência, que já aprendemos. Esse passo é feito para verificar se a variável tem potencial para participar do modelo final. Seguem os comandos:

Vamos **ilustrar esse cenário** com a base de dados de **leucemia aguda**. Para isso, vamos **carregar a base de dados**:


Em muitas aplicações, algumas covariáveis apresentam valores muito elevados numericamente, como renda, população, carga viral, PIB ou número de acessos. Nesses casos, é comum utilizar a transformação logarítmica da variável devido às seguintes vantagens:

- Redução da magnitude numérica dos valores, tornando a escala mais controlada;
- Melhora da estabilidade computacional dos algoritmos de estimação;
- Redução de problemas de convergência em métodos iterativos;
- Melhor condicionamento das matrizes envolvidas na estimação;
- Diminuição da influência de valores extremamente altos;
- Possibilidade de interpretar os efeitos em termos percentuais ou multiplicativos;
- Redução de assimetria em variáveis fortemente assimétricas.

Assim, a transformação logarítmica frequentemente produz modelos mais estáveis, interpretáveis e adequados aos dados observados.

Para esta base de dados, consideraremos a seguinte estrutura regressora:
$$\beta_0+\beta_1x_i,$$
em que $x_i$ é o valor de log10\_wbc para o $i$ paciente, com $i=1,...,17$.

**MODELO EXPONENCIAL**

Para o modelo exponencial, tem-se a seguinte saída:

```{r}
tidy(ajuste_exp)
```

De **acordo com o modelo ajustado, temos que**: 

* `rate` = $exp\{\hat{\beta_0}\}=0,03\rightarrow \hat{\beta_0} = ln(0,03) = -3.51$ 
* $\hat{\beta_1}=0,633$
* $\hat{\beta_2}=1,07$

Portanto, cada tempo de sobrevivência $T_i$ possui a seguinte distribuição:

$$T_i\sim Exp(\hat{\alpha_i}=e^{-3,51 + 0,633x_{i1}+ 1,07x_{i2}})$$

Com essa distribuição estimada, podemos obter todas as quantidades de interesse em sobrevivência (funções básicas e medidas-resumo)

Nesse modelo:

* A estimativa de $\hat{\beta_0}$ é negativa
* A estimativa de $\hat{\beta_1}$ é positiva
* A estimativa de $\hat{\beta_2}$ é positiva
* Pelo teste de Wald, há evidências de que $\beta_2$ é diferente de 0.


Podemos concluir que não há diferença na sobrevivência dos ratos comparando o grupo 2 com o grupo 1. Com respeito ao grupo 3, podemos interpretar o coeficiente $\hat{\beta_2}$ da seguinte forma:


**TESTE DA RAZÃO DE VEROSSIMILHANÇA**

Para investigar a adequação do modelo mais complexo (gama generalizado) comparado aos demais a partir do teste TRV, considere os seguintes comandos:

```{r}
TRV_exp = 2*(ajuste_gengama$loglik-ajuste_exp$loglik)
P_valor_exp = pchisq(TRV_exp,2,lower.tail=F)
TRV_wei = 2*(ajuste_gengama$loglik-ajuste_wei$loglik)
P_valor_wei = pchisq(TRV_wei,1,lower.tail=F)
TRV_lognorm = 2*(ajuste_gengama$loglik-ajuste_lognorm$loglik)
P_valor_lognorm = pchisq(TRV_lognorm,1,lower.tail=F)
TRV_gama = 2*(ajuste_gengama$loglik-ajuste_gama$loglik)
P_valor_gama = pchisq(TRV_gama,1,lower.tail=F)
Comparacao<-c("Exponencial x GG", "Weibull x GG",
           "LogNormal x GG",
           "Gama x GG")
P_valor = c(P_valor_exp,P_valor_wei,
            P_valor_lognorm,P_valor_gama)
TRV<-data.frame(Comparacao,P_valor)
TRV
```

De acordo com os valores-p do teste TRV, nenhum modelo mais simples é preferível para o ajuste aos dados.
